Basic HTML Tags

You learned the following about basic HTML tags in this module:

line break

[cite_start]

The br [cite: 2] [cite_start]is used to create line breaks in code[cite: 3].

[cite_start]

Line breaks are commonly used for mailing addresses and should not be used for formatting a web page[cite: 3].

[cite_start]

CSS [cite: 4][cite_start], which is discussed later in the course, should be used for formatting a web page[cite: 4].

whitespace

[cite_start]

Whitespace is the unprinted character created by hitting the space bar[cite: 5].

[cite_start]

No matter how many spaces are entered, the browser recognizes it as one space[cite: 6].

sections

[cite_start]

A section is a collection of related paragraphs, images, or other content that includes a heading[cite: 7].

[cite_start]

You can have sections inside sections inside sections[cite: 8]. [cite_start]They are a **container tag** [cite: 8] used to group content together.

headings

[cite_start]

A heading provides a title for a section of content[cite: 9].

[cite_start]

There are six levels of headings[cite: 10]:

  1. h1
  2. h2
  3. h3
  4. h4
  5. h5
  6. h6
[cite_start]

The **h1** [cite: 11] [cite_start]tag is the first level heading and should describe the contents of the entire page[cite: 11].

[cite_start]

It **MUST be used first** [cite: 12] [cite_start]in your document, before any other heading level[cite: 12].

[cite_start]

In this assignment, the **h1** [cite: 13] [cite_start]tag is at the top of the page and contains the content "Module 2 Notes - HTML Common Tags"[cite: 13].

[cite_start]

On a real site, your h1 [cite: 14] [cite_start]and title would normally contain the same text[cite: 14].

[cite_start]

The h2 through h6 [cite: 15] [cite_start]tags should be used for subtopics/subchapters[cite: 15].

[cite_start]

You may not skip heading levels [cite: 16][cite_start], and headings should **not be used for style**[cite: 16].

[cite_start]

By the time you are done coding this assignment, you should have a better understanding of heading tags[cite: 17].

Special Characters

[cite_start]

Entities are used for coding special characters[cite: 21]. [cite_start]While there are three ways to code entities, it is **recommended** [cite: 21] [cite_start]that you use the entity name[cite: 21].

[cite_start]

It is the easiest to remember[cite: 22]. [cite_start]You can learn more about character entities at w3schools.com[cite: 22].

[cite_start]

The most common characters that you will use in this course are those for the trademark symbol and the copyright symbol[cite: 23].

[cite_start]

Non-breaking spaces should only be used to keep two words together [cite: 24][cite_start], so one word does not wrap to the next line[cite: 24].

[cite_start]

For example, **Paradise Bakery** [cite: 25] [cite_start]would probably want those two words to appear together at all times[cite: 25].

[cite_start]

They would never want the word Paradise on one line and then have the word Bakery wrapped to the next line[cite: 26].

[cite_start]

To keep the words together, you would code a non-breaking space between the two words so it would be treated as one long word[cite: 27].

Lists

[cite_start]

The two most common types of lists are ordered lists and unordered Lists [cite: 28]

Ordered Lists

[cite_start]

Ordered lists should be used for lists that require an order[cite: 29].

[cite_start]

The steps needed to enroll in a course at ASU might be one example of an ordered list[cite: 29].

[cite_start]

There are many steps, and they need to be done in a **specific order**[cite: 30].

[cite_start]

Ordered lists would also be used for a list where the items are ranked[cite: 31].

Unordered Lists

[cite_start]

Unordered lists should be used for lists that **do not require order**[cite: 32].

An unordered list was used above for the various text formatting tags.
